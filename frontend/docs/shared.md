# Shared — Frontend Common Library (@/shared)

Shared is a **reusable library** of cross-cutting frontend concerns.

It is imported by any feature that needs UI primitives, API access, global state, types, or utilities — without duplicating code.

---

# Core Idea

Shared = one place for everything used across multiple features

```text
@/shared → any feature module
```

---

# What Shared Contains

| Folder       | Purpose                                                 |
| ------------ | ------------------------------------------------------- |
| `components` | reusable UI components built on top of shadcn           |
| `constants`  | app-wide constant values                                |
| `hooks`      | reusable React hooks                                    |
| `lib`        | third-party client setup (axios, react-query)           |
| `stores`     | global Zustand state slices                             |
| `types`      | shared generic TypeScript types                         |
| `ui`         | raw shadcn/ui primitives (auto-generated, never edited) |

---

# File Structure

```text
src/
  shared/
    components/       ← composite reusable components
    constants/        ← app-wide constant values
    hooks/            ← reusable React hooks
    lib/
      api.ts          ← axios instance
      query-client.ts ← react-query client
    stores/
      ui.slice.ts     ← UI state slice
    types/
      paginated-result.type.ts
    ui/               ← shadcn primitives (never edit manually)
    index.ts          ← single barrel export
```

---

# Barrel Export — index.ts

Everything is re-exported from a single entry point:

```ts
export * from './components';
export * from './constants';
export * from './hooks';
export * from './lib';
export * from './stores';
export * from './types';
export * from './ui';
```

Usage anywhere in the app:

```ts
import {
  api,
  queryClient,
  useMediaQuery,
  PAGINATION_DEFAULTS,
  type PaginatedResult,
} from '@/shared';
```

# components

## Purpose

Composite reusable UI components built on top of shadcn primitives.

The difference from `ui/`:

|          | `ui/`                        | `components/`                           |
| -------- | ---------------------------- | --------------------------------------- |
| Source   | auto-generated by shadcn CLI | hand-written                            |
| Edit?    | ❌ never                     | ✅ yes                                  |
| Examples | `Button`, `Input`, `Dialog`  | `PageHeader`, `DataTable`, `EmptyState` |

## Rules

✔ A component belongs here when it is used in **2+ features**  
✔ Must be purely presentational — no business logic, no API calls  
✔ Feature-specific components stay inside the feature folder  
✔ Always built from `ui/` primitives, never from raw HTML

# constants

## Purpose

App-wide constant values that are referenced in multiple places.

Avoids magic numbers and strings scattered across the codebase.

## Pattern

```ts
export const [SCREAMING_SNAKE_CASE] = {
  field: value,
} as const;
```

## Example

```ts
export const PAGINATION_DEFAULTS = {
  page: 1,
  limit: 20,
} as const;
```

## Usage

```ts
import { PAGINATION_DEFAULTS } from '@/shared';

const [page, setPage] = useState(PAGINATION_DEFAULTS.page);
```

## Rules

✔ Always use `as const` — prevents accidental mutation and narrows types  
✔ A value belongs here when it appears in **2+ files** across different features  
✔ Feature-specific constants (e.g. route paths for one module) stay inside the feature

# hooks

## Purpose

Reusable React hooks that encapsulate browser APIs, side effects, or derived state that is needed in multiple features.

## Pattern

```ts
export function use[Name]([params]): [ReturnType] {
  // setup state and effects
  return value;
}
```

## Example — `useMediaQuery`

Subscribes to a CSS media query and returns whether it currently matches.

```ts
export function useMediaQuery(query: string): boolean {
  const [matches, setMatches] = useState(
    () => window.matchMedia(query).matches
  );

  useEffect(() => {
    const media = window.matchMedia(query);
    const handler = (e: MediaQueryListEvent) => setMatches(e.matches);
    media.addEventListener('change', handler);
    return () => media.removeEventListener('change', handler);
  }, [query]);

  return matches;
}
```

## Usage

```ts
const isMobile = useMediaQuery('(max-width: 768px)');
```

## Rules

✔ Hooks in shared must be generic — no entity names, no feature-specific logic  
✔ A hook belongs here when used in **2+ features**  
✔ Must clean up side effects in the `useEffect` return  
✔ Feature-specific hooks (e.g. `useTodoFilters`) stay inside the feature

# lib

## Purpose

Configuration and setup of third-party clients used globally across the app.

One file per external dependency. Nothing else lives here.

---

## `api.ts` — Axios Instance

Single configured axios instance used for all HTTP requests.

```ts
export const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL ?? 'http://localhost:3000/api/v1',
  headers: { 'Content-Type': 'application/json' },
});
```

Also unwraps the backend envelope automatically:

```text
{ data: {...}, statusCode: 200, timestamp: "..." }  →  {...}
```

So feature code always receives the payload directly — never the wrapper.

### Rules

✔ Always import `api` from `@/shared` — never create a second axios instance  
✔ All interceptors (auth headers, error handling, envelope unwrap) go here  
✔ Never use `fetch` or raw `axios` in feature code

---

## `query-client.ts` — React Query Client

Single `QueryClient` instance with app-wide defaults.

```ts
export const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 1000 * 60 * 5, // 5 minutes
      retry: 1,
    },
  },
});
```

| Option      | Value  | Meaning                                       |
| ----------- | ------ | --------------------------------------------- |
| `staleTime` | `5min` | cached data stays fresh for 5 minutes         |
| `retry`     | `1`    | retries a failed request once before erroring |

### Rules

✔ Always pass `queryClient` to `<QueryClientProvider>` in `main.tsx`  
✔ Never create a second `QueryClient` in a feature  
✔ Override defaults per-query when needed via `useQuery` options

# stores

## Purpose

Global Zustand state slices for UI or app-level state that is shared across features.

State that belongs to a single feature stays inside that feature.

## Slice Pattern

Each concern gets its own slice file:

```ts
// [name].slice.ts
export interface [Name]Slice {
  // state fields
  // action signatures
}

export const create[Name]Slice: StateCreator<[Name]Slice> = (set) => ({
  // initial state
  // action implementations
});
```

## Example — `UiSlice`

Manages global UI state (sidebar, modals, theme, etc.):

```ts
export interface UiSlice {
  sidebarOpen: boolean;
  toggleSidebar: () => void;
}

export const createUiSlice: StateCreator<UiSlice> = (set) => ({
  sidebarOpen: true,
  toggleSidebar: () => set((s) => ({ sidebarOpen: !s.sidebarOpen })),
});
```

## Combining Slices into a Store

```ts
// stores/index.ts
import { create } from 'zustand';

export const useStore = create<UiSlice>()((...a) => ({
  ...createUiSlice(...a),
}));
```

## Usage

```ts
const { sidebarOpen, toggleSidebar } = useStore();
```

## Rules

✔ One slice per concern — never put unrelated state in the same slice  
✔ Shared stores only for state needed in **2+ features**  
✔ Feature-specific state (e.g. selected todo id) stays in the feature  
✔ Never store server data here — that belongs in React Query cache

# types

## Purpose

Shared generic TypeScript types and interfaces used as return types across API calls, hooks, and components.

Add here any type that appears in **2+ files** across different features.

## `PaginatedResult<T>`

Standard shape returned by all paginated API endpoints.

```ts
export interface PaginatedResult<T> {
  items: T[];
  total: number;
  page: number;
  limit: number;
  totalPages: number;
}
```

## Usage

```ts
// API call return type
const { data } = useQuery<PaginatedResult<Todo>>(...);

// Component prop type
interface DataTableProps<T> {
  data: PaginatedResult<T>;
}
```

✔ Always import as a type: `import type { PaginatedResult } from '@/shared'`  
✔ Never create entity-specific variants — `PaginatedTodos` is wrong, `PaginatedResult<Todo>` is correct

# ui

## Purpose

Raw shadcn/ui primitives — `Button`, `Input`, `Dialog`, `Table`, `Card`, and so on.

This folder is **managed by the shadcn CLI** and must never be edited manually.

```bash
# Add a new primitive
npx shadcn@latest add button
```

## Rules

✔ Never edit files in `ui/` — changes will be overwritten on next `shadcn add`  
✔ To customize a component, wrap it in `components/` and extend it there  
✔ Import primitives from `@/shared` — never from deep paths
